# Dashboard Guide

## Overview

The CloudWatch dashboard provides a real-time view of the application’s health and performance.

Dashboard Name:

ObservableServiceDashboard

The dashboard visualizes both application metrics and infrastructure metrics.

---

## Key Metrics

The dashboard displays the following metrics:

RequestCount  
ErrorCount  
OrdersCreated  
OrderValue  
ApiLatency
CPUUtilization

These metrics help engineers understand system activity and detect abnormal behavior.

---

## Golden Signals

The dashboard partially implements the Golden Signals monitoring model.

Traffic  
Measured using RequestCount

Errors  
Measured using ErrorCount

Saturation  
Measured using CPUUtilization

Latency  
Observed through application logs (latency_ms field)

---

## Widget Explanation

### RequestCount

Shows the number of API requests received over time.

Used to monitor traffic spikes.

---

### ErrorCount

Shows the number of application errors.

If this metric spikes, the HighErrorRate alarm may trigger.

---

### OrdersCreated

Represents the number of simulated orders created.

Used as a business metric.

---

### OrderValue

Tracks the value of orders generated by the API.

Helps demonstrate business observability.

---

### ApiLatency
Displays API response time in milliseconds.
Higher values indicate slower request processing.
This widget helps detect latency spikes and performance degradation.

---
### CPUUtilization

Shows the EC2 instance CPU usage.

High CPU may indicate resource saturation.

---

## Troubleshooting Using the Dashboard

When an incident occurs:

1. Check the RequestCount graph to detect traffic spikes
2. Check ErrorCount to see if errors increased
3. Review CPUUtilization to identify resource issues
4. Investigate logs in CloudWatch Logs for detailed errors
